الدرس 01

مقدمة عن HTML

ما هي لغة HTML؟ ولماذا نستخدمها لبناء صفحات الويب؟

تعريف بسيط

HTML اختصار لـ HyperText Markup Language، وهي لغة توصيف (وليست لغة برمجة) تُستخدم لبناء هيكل صفحات الويب. بمعنى آخر: نستخدم HTML لنقول للمتصفح "هذا عنوان"، "هذه فقرة"، "هذه صورة"، "هذا رابط".

كل صفحة تفتحها على الإنترنت — مثل Google أو YouTube — مبنيّة باستخدام HTML كأساس.

لماذا نستخدم HTML؟

  • لبناء هيكل الصفحة وتنظيم محتواها (نصوص، صور، روابط، جداول...).
  • لأنها اللغة الأساسية للويب، وكل المتصفحات تفهمها مباشرة.
  • سهلة التعلم وتُعتبر أول خطوة في طريق تعلم تطوير الويب.
  • تعمل مع CSS (للتنسيق) وJavaScript (للتفاعل).

ما هو "التاج" (Tag)؟

عناصر HTML تُكتب على شكل تاجات (Tags) محاطة بعلامتي أصغر وأكبر < >. معظم التاجات تأتي بشكل زوجي: تاج فتح وتاج إغلاق. 

<p>هذه فقرة نصية</p>

هنا <p> هو تاج الفتح، و</p> هو تاج الإغلاق، والنص بينهما هو محتوى العنصر.

محرر تجريبي — جرّب بنفسك

💡 ملاحظات للامتحان

  • HTML ليست لغة برمجة، بل لغة توصيف (Markup).
  • كل ملف HTML يبدأ غالباً بـ <!DOCTYPE html>.
  • تكتب التاجات بأحرف صغيرة (مع أنها لا تفرّق بين الحالتين).
الدرس 02

هيكل الصفحة الأساسي

التعرّف على التاجات الأساسية: <html>, <head>, <title>, <body>.

الهيكل العام لأي صفحة HTML

أي صفحة HTML احترافية تتكوّن من نفس الهيكل العام التالي:

<!DOCTYPE html>
<html>
  <head>
    <title>عنوان الصفحة</title>
  </head>
  <body>
    <!-- محتوى الصفحة هنا -->
  </body>
</html>
<head>

يحتوي على معلومات عن الصفحة ولا تظهر للمستخدم مباشرة، مثل العنوان والروابط الخارجية وملفات التنسيق.

متى نستخدمه؟ دائماً — في كل صفحة HTML، يأتي قبل <body>.
<head>
  <title>صفحتي</title>
</head>
<title>

يحدّد عنوان الصفحة الذي يظهر في تبويب المتصفح وفي نتائج البحث في Google.

متى نستخدمه؟ داخل <head>، مرة واحدة لكل صفحة.
<title>متجر الكتب</title>
<body>

يحتوي على المحتوى الظاهر للمستخدم: النصوص، الصور، الأزرار، الروابط، وكل ما تراه في الصفحة.

متى نستخدمه؟ في كل صفحة، يأتي بعد <head>.
<body>
  <h1>مرحباً</h1>
</body>
<html>

العنصر الجذري الذي يحتوي كل عناصر الصفحة. يأتي بعد <!DOCTYPE html> مباشرة.

متى نستخدمه؟ دائماً، مرة واحدة كحاوية أساسية للصفحة.
<html>
  <head>...</head>
  <body>...</body>
</html>
محرر تجريبي — هيكل صفحة كامل

💡 ملاحظات للامتحان

  • <head> لا يظهر للمستخدم، بينما <body> يظهر.
  • <title> يجب أن يكون داخل <head>.
  • كل صفحة لها <body> واحد فقط.
الدرس 03

التعامل مع النصوص

تاجات النصوص الأساسية: <p>, <br>, <i>, <u>, والعناوين <h1> إلى <h6>.

العناوين (Headings)

في HTML لدينا ستة مستويات من العناوين، من <h1> (الأكبر والأهم) إلى <h6> (الأصغر). 

<h1>عنوان رئيسي</h1>
<h2>عنوان فرعي</h2>
<h3>عنوان أصغر</h3>
<p>

يُستخدم لكتابة فقرة نصية. كل <p> يأتي في سطر جديد ويترك مسافة قبله وبعده.

متى نستخدمه؟ لأي نص عادي على شكل فقرة.
<p>أحب تعلّم البرمجة.</p>
<br>

يُستخدم للنزول إلى سطر جديد داخل النص. تاج مفرد (لا يحتاج إغلاق).

متى نستخدمه؟ عند الحاجة إلى كسر السطر داخل فقرة (مثل العناوين البريدية أو الشعر).
السطر الأول<br>السطر الثاني
<i>

يجعل النص مائلاً (Italic). غالباً يُستخدم لإبراز كلمة أجنبية أو مصطلح.

متى نستخدمه؟ للتمييز البصري للنص دون تغيير معناه.
كلمة <i>مائلة</i> هنا
<u>

يضع خطاً تحت النص (Underline).

متى نستخدمه؟ لإبراز كلمة بالخط السفلي. (تجنّب استخدامه على نصوص ليست روابط لتفادي التشويش.)
كلمة <u>مهمة</u> جداً
<b> / <strong>

يجعل النص غامقاً (Bold). الفرق: <strong> يدل على أهمية النص دلالياً.

متى نستخدمه؟ لتمييز كلمة مهمة في الجملة.
هذا <b>مهم</b> جداً
<hr>

يرسم خطاً أفقياً فاصلاً. تاج مفرد بدون إغلاق.

متى نستخدمه؟ للفصل بين أقسام مختلفة في الصفحة.
قسم 1<hr>قسم 2
محرر تجريبي — تنسيق النصوص

💡 ملاحظات للامتحان

  • <br> تاج مفرد ولا يحتاج إغلاق.
  • لا يجوز استخدام أكثر من <h1> واحد للصفحة بشكل عام.
  • <i> = مائل، <u> = تحته خط، <b> = غامق.
  • المسافات الزائدة وأسطر الفراغ في الكود لا تظهر في المتصفح — استخدم <br> أو <p>.
الدرس 04

الروابط (Hyperlinks)

شرح كامل لتاج <a> وأنواع الروابط وخصائصها.

ما هو الرابط؟

الرابط (Hyperlink) هو نص أو صورة يمكن النقر عليها للانتقال إلى صفحة أخرى، أو إلى ملف، أو إلى موقع آخر على الإنترنت. نستخدم له تاج <a> (اختصاراً لكلمة anchor). 

<a href="https://google.com">اذهب إلى Google</a>

الخصائص (Attributes) المهمة

الخاصية
الوصف
مثال
href
تحدد وجهة الرابط (إلى أين سننتقل عند الضغط).
href="page.html"
target
تحدد طريقة فتح الرابط: نفس النافذة أم نافذة جديدة.
target="_blank"
title
نص يظهر عند تمرير الفأرة فوق الرابط (Tooltip).
title="موقع جوجل"

قيم خاصية target الشائعة:

  • _self — يفتح الرابط في نفس النافذة (الافتراضي).
  • _blank — يفتح الرابط في نافذة/تبويب جديد.

أنواع الروابط

1

رابط خارجي

ينقلك إلى موقع آخر على الإنترنت. يبدأ بـ http:// أو https://.

<a href="https://youtube.com">
  يوتيوب
</a>
2

رابط داخلي

ينقلك إلى صفحة أخرى داخل نفس الموقع. تكتب اسم الملف فقط.

<a href="about.html">
  من نحن
</a>
3

رابط لملف

ينزّل ملفاً عند الضغط (مثل PDF أو صورة). يكفي وضع مسار الملف في href.

<a href="cv.pdf">
  تحميل السيرة الذاتية
</a>
4

رابط داخل نفس الصفحة

ينقلك إلى قسم محدد من الصفحة عبر id.

<a href="#contact">
  اتصل بنا
</a>
5

رابط بريد إلكتروني

يفتح برنامج البريد. نستخدم mailto:.

<a href="mailto:a@b.com">
  راسلني
</a>

💡 ملاحظات للامتحان

  • تاج الرابط هو <a> — أي وجهة دون href لن تعمل.
  • target="_blank" يفتح نافذة جديدة.
  • الفرق بين الرابط الداخلي والخارجي: الخارجي يبدأ بـ http://، الداخلي يستخدم اسم الملف فقط.
  • أخطاء شائعة: نسيان وضع علامتي التنصيص حول قيمة href، أو نسيان إغلاق التاج </a>.
الدرس 05

القوائم (Lists)

القوائم المرتبة وغير المرتبة: <ul>, <ol>, <li>.

أنواع القوائم في HTML

القوائم تساعدنا في تنظيم المعلومات على شكل بنود. لدينا نوعان أساسيان:

  • قائمة غير مرتبة (Unordered List) — بنود بنقاط •.
  • قائمة مرتبة (Ordered List) — بنود مرقمة 1, 2, 3...
<ul>

قائمة غير مرتبة (Unordered) — كل بند يبدأ بنقطة.

متى نستخدمها؟ عندما لا يهم ترتيب البنود (مثل قائمة هوايات).
<ul>
  <li>تفاح</li>
  <li>موز</li>
</ul>
<ol>

قائمة مرتبة (Ordered) — كل بند يأخذ رقماً تلقائياً.

متى نستخدمها؟ عندما يهم الترتيب (مثل خطوات وصفة طبخ).
<ol>
  <li>اغسل التفاح</li>
  <li>قطّعه</li>
</ol>
<li>

يمثّل عنصراً واحداً (List Item) داخل القائمة. يُستخدم داخل <ul> أو <ol>.

متى نستخدمه؟ لكل بند داخل أي قائمة.
<li>بند واحد</li>

الفرق بين <ul> و <ol>

<ul> — غير مرتبة

  • قهوة
  • شاي
  • عصير

البنود تظهر بنقاط •

<ol> — مرتبة

  1. قهوة
  2. شاي
  3. عصير

البنود تظهر بأرقام 1, 2, 3

محرر تجريبي — القوائم

💡 ملاحظات للامتحان

  • كل بند في القائمة يجب أن يكون داخل <li>.
  • يمكنك وضع قائمة داخل قائمة أخرى (Nested Lists).
  • ul = unordered (نقاط)، ol = ordered (أرقام)، li = list item.
  • خطأ شائع: نسيان <li> ووضع النص مباشرة داخل <ul>.
الدرس 06

الجداول (Tables) مهم جداً للامتحان

شرح كامل لـ <table>, <tr>, <td>, <th> + خاصيتي rowspan و colspan.

الجدول في HTML

الجدول مكوّن من صفوف (Rows) وخلايا (Cells). كل خلية تنتمي إلى صف، وكل صف يحتوي على عدّة خلايا.

<table>

يُحدّد الجدول ككل، وهو الحاوية الأساسية.

متى نستخدمه؟ لعرض البيانات في صفوف وأعمدة.
<tr>

Table Row — يُمثّل صفّاً واحداً داخل الجدول.

متى نستخدمه؟ لكل صف أفقي.
<th>

Table Header — خلية عنوان (تظهر غامقة وفي المنتصف).

متى نستخدمه؟ لعناوين الأعمدة في الصف الأول عادة.
<td>

Table Data — خلية بيانات عادية.

متى نستخدمه؟ لمحتوى الجدول الفعلي (الأرقام، الأسماء، إلخ).

مثال جدول بسيط

<table border="1">
  <tr>
    <th>الاسم</th>
    <th>العمر</th>
  </tr>
  <tr>
    <td>أحمد</td>
    <td>25</td>
  </tr>
  <tr>
    <td>سارة</td>
    <td>22</td>
  </tr>
</table>
الاسمالعمر
أحمد25
سارة22

🔥 دمج الخلايا: colspan و rowspan

هاتان خاصيتان مهمّتان جداً تُستخدمان لـدمج خلايا الجدول.

colspan — دمج عمودين أو أكثر

تُستخدم لجعل خلية تمتد أفقياً عبر عدة أعمدة.

<tr>
  <td colspan="2">خلية ممتدة</td>
</tr>
المادةالدرجة
المجموع: 95
رياضيات50
علوم45

لاحظ كيف "المجموع" يمتد على عمودين.

rowspan — دمج صفّين أو أكثر

تُستخدم لجعل خلية تمتد عمودياً عبر عدة صفوف.

<tr>
  <td rowspan="2">خلية ممتدة</td>
  <td>A</td>
</tr>
<tr>
  <td>B</td>
</tr>
الصفالمادةالمعلم
العاشررياضياتأ. أحمد
علومأ. سارة

لاحظ كيف "العاشر" يمتد على صفّين.

انتبه: عند استخدام colspan="2" في صف، يجب أن يحتوي الصف على خلية واحدة أقل من باقي الصفوف لأن خلية واحدة تأخذ مساحة عمودين.
محرر تجريبي — جدول مع colspan و rowspan

💡 ملاحظات للامتحان

  • <th> للعناوين، <td> للبيانات.
  • colspan = دمج أعمدة (أفقي). rowspan = دمج صفوف (عمودي).
  • border="1" يضيف حدوداً للجدول، أما الأنسب فهو استخدام CSS.
  • تذكّر: عدد <td> في كل <tr> يجب أن يساوي عدد الأعمدة (مع مراعاة الدمج).
الدرس 07

النماذج (Forms)

كيفية إنشاء حقول إدخال لجمع البيانات من المستخدم.

ما هو النموذج؟

النموذج (Form) هو وسيلة لجمع البيانات من المستخدم — مثل صفحات تسجيل الدخول، التسجيل في موقع، أو ترك تعليق. نستخدم تاج <form> كحاوية، وتحته نضع حقول الإدخال (<input>).

<form>

حاوية النموذج. تجمع كل حقول الإدخال معاً.

متى نستخدمه؟ عند الحاجة لجمع بيانات من المستخدم.
<input>

حقل إدخال — يتغيّر نوعه حسب type: text, password, email, checkbox, radio, submit...

متى نستخدمه؟ لكل حقل بيانات (اسم، إيميل، كلمة سر، إلخ).
<label>

يكتب التسمية أمام الحقل (مثل "الاسم:"). يُربط بالحقل عبر for.

متى نستخدمه؟ دائماً قبل كل حقل لتحسين تجربة المستخدم.
<textarea>

حقل نصّي كبير ومتعدد الأسطر (مثل خانة الرسائل).

متى نستخدمه؟ عندما يحتاج المستخدم لكتابة نص طويل.
<button>

زر — غالباً يُستخدم لإرسال النموذج.

متى نستخدمه؟ في نهاية النموذج لإرسال البيانات.
<select>

قائمة منسدلة (Drop-down) فيها خيارات <option>.

متى نستخدمه؟ لاختيار قيمة واحدة من قائمة (مثل اختيار الدولة).
محرر تجريبي — نموذج كامل

💡 ملاحظات للامتحان

  • <input> تاج مفرد، نوعه يحدد شكله.
  • أنواع type الأكثر شيوعاً: text, password, email, radio, checkbox, submit.
  • اربط <label> بالحقل باستخدام for + id.
  • أزرار radio التي تشترك في نفس name = اختيار واحد فقط منها.
الدرس 08

الوسائط (Media)

إضافة الصور <img>، الصوت <audio>، والفيديو <video>.

<img>

يُستخدم لعرض الصور. تاج مفرد. أهم خاصياته: src (مسار الصورة)، alt (نص بديل)، width و height.

<img src="cat.jpg" 
     alt="قطة" 
     width="200">
<audio>

يضيف ملف صوتي مع أزرار التشغيل. استخدم controls لإظهار الأدوات.

<audio src="song.mp3" 
       controls>
</audio>
<video>

يضيف مقطع فيديو. خصائص شبيهة بالصوت + width و height.

<video src="film.mp4" 
       controls 
       width="320">
</video>
<iframe>

يُضمّن صفحة أخرى داخل صفحتك (مثل خرائط Google أو فيديو يوتيوب).

<iframe 
  src="https://example.com">
</iframe>
محرر تجريبي — الصور والفيديو والصوت

💡 ملاحظات للامتحان

  • <img> تاج مفرد ولا يحتاج إغلاق.
  • alt مهم للوصولية ولظهور النص إذا فشل تحميل الصورة.
  • أضف controls ليظهر زر التشغيل في الصوت والفيديو.
الدرس 09

Semantic HTML

تاجات ذات معنى تساعد المتصفحات ومحركات البحث على فهم بنية الصفحة.

ما معنى Semantic؟

Semantic = دلالي. التاجات الدلالية هي تاجات اسمها يصف وظيفتها. بدل استخدام <div> لكل شيء، نستخدم تاجات أوضح مثل <header> و <footer> و <article>.

<header>

رأس الصفحة — يحتوي عادة على الشعار والقائمة العلوية.

<nav>

قائمة التنقل — مجموعة الروابط الرئيسية في الموقع.

<main>

المحتوى الأساسي للصفحة. مرة واحدة فقط لكل صفحة.

<section>

قسم له موضوع واحد، مثل قسم المنتجات أو قسم الفريق.

<article>

محتوى مستقل، مثل مقال أو منشور — يمكن قراءته بمفرده.

<footer>

أسفل الصفحة — يحتوي عادة على حقوق الملكية والروابط الجانبية.

محرر تجريبي — هيكل دلالي

💡 ملاحظات للامتحان

  • استخدام التاجات الدلالية يحسّن SEO والوصولية.
  • <main> يجب أن يكون مرة واحدة فقط لكل صفحة.
  • الفرق بين <section> و <article>: المقال مستقل بذاته.
الدرس 10

Best Practices + الخصائص (Attributes)

شرح id, class, align, href + أفضل الممارسات.

الخصائص (Attributes) — مهم جداً

الخاصية هي معلومة إضافية نضيفها للتاج لتغيير سلوكه أو شكله. تُكتب داخل تاج الفتح بصيغة اسم="قيمة".

الوصف والاستخدام
مثال
id
يعطي العنصر معرّف فريد داخل الصفحة. لا يمكن أن يتكرر. يُستخدم للتنسيق بـ CSS، أو للروابط الداخلية، أو للوصول من JS.
<p id="intro">
class
تعطي العنصر اسم فئة يمكن أن يتكرر على عدة عناصر. تستخدم للتنسيق بـ CSS بشكل جماعي.
<p class="note">
href
تُستخدم مع <a> فقط — تُحدّد وجهة الرابط.
<a href="page.html">
align
تُحدّد محاذاة المحتوى: left, center, right. (قديمة، يُفضّل استخدام CSS بدلاً منها لكنها قد تأتي في الامتحان.)
<p align="center">
src
تُستخدم مع الصور والفيديو لتحديد مصدر الملف.
<img src="cat.png">
alt
نص بديل يظهر إذا لم تُحمّل الصورة (وللوصولية).
<img alt="قطة">
target
تُحدّد طريقة فتح الرابط: _blank = نافذة جديدة، _self = نفس النافذة.
<a target="_blank">

الفرق بين id و class

id (معرّف)

  • فريد — مرة واحدة في الصفحة.
  • يستخدم للروابط الداخلية #id.
  • في CSS: #myId { ... }

class (فئة)

  • يتكرر — يمكن استخدامه على عدة عناصر.
  • للتنسيق المتشابه لمجموعة عناصر.
  • في CSS: .myClass { ... }

أفضل الممارسات (Best Practices)

  1. استخدم تاجات صغيرة دائماً (<p> بدل <P>).
  2. أغلق كل التاجات حتى لو سمح المتصفح بترك بعضها.
  3. استخدم alt لكل صورة.
  4. اعتمد على التاجات الدلالية بدل <div> للجميع.
  5. استخدم CSS للتنسيق وليس خصائص HTML القديمة (مثل align, bgcolor).
  6. أعطِ صفحتك <title> واضحاً ومفيداً.
  7. أبقِ الكود مرتباً ومنسّقاً (Indentation).

الامتحان الشامل في HTML

مزيج من الأسئلة النظرية (اختيار من متعدد) والأسئلة البرمجية (اكتب الكود وتأكّد من صحّته في المعاينة مباشرة). تُحسب الدرجة فوراً عند الإنهاء.

📝 18 سؤال نظري
💻 6 أسئلة برمجية
⏱️ ~ 30 دقيقة
🎯 النجاح من 50%

💡 إرشادات قبل البدء

  • أسئلة الاختيار من متعدد: إجابة واحدة فقط ولا يمكن تغييرها بعد الاختيار.
  • الأسئلة البرمجية: اكتب الكود ثم اضغط "تحقق من الإجابة" — يمكنك المحاولة عدة مرات.
  • زر "تشغيل" يعرض الكود في المعاينة، وزر "إظهار الحل" يكشف الإجابة المقترحة.
  • بعد الانتهاء، اضغط "إنهاء وعرض النتيجة" لمشاهدة درجتك وتقييمك.
